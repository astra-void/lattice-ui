## Install

### Global CLI

```bash
lattice add system
```

### Monorepo Local Script

```bash
pnpm lattice add system
```

## Package

- npm: `@lattice-ui/system`
- peer dependencies: `@rbxts/react`, `@rbxts/react-roblox`
- internal dependencies: `@lattice-ui/core`, `@lattice-ui/style`

## Primary Exports

- `SystemProvider`
- `useSystemTheme`
- `DensityProvider`
- `useDensity`
- `Stack`
- `Row`
- `Grid`
- `Surface`
- `surface token helpers`

## Usage Snippet

```tsx
import { Row, Stack, Surface, SystemProvider } from "@lattice-ui/system";
import { Text } from "@lattice-ui/style";

export function SettingsLayout() {
  return (
    <SystemProvider>
      <Stack gap="md" padding="md">
        <Surface tone="elevated">
          <Text>Profile panel</Text>
        </Surface>
        <Row gap="sm">
          <Surface tone="surface">
            <Text>Left</Text>
          </Surface>
          <Surface tone="surface">
            <Text>Right</Text>
          </Surface>
        </Row>
      </Stack>
    </SystemProvider>
  );
}
```

## Start Here for Design Guide

`system` defines app-level layout/runtime context. For the combined design workflow with
`style` tokens and recipes, start with
[Design with Style + System](/getting-started/design-with-style-system/).

## Notes

- `SystemProvider` and density/theme helpers provide app-level UI context.
- Layout primitives (`Stack`, `Row`, `Grid`) and `Surface` compose base screen scaffolding.
