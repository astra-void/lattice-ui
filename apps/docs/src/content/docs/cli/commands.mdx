## Global Options

- `--help`
- `--version`

## `create`

Create a new project from the rbxts template.

Signature:

`lattice create [project-path] [--yes] [--pm <pnpm|npm|yarn>] [--git] [--template rbxts] [--lint] [--no-lint]`

Behavior:

- `[project-path]` is optional in interactive mode.
- In `--yes` mode, `[project-path]` is required.
- `--lint` and `--no-lint` are mutually exclusive.
- `--template` currently supports only `rbxts`.
- Generated projects include `@lattice-ui/cli` in `devDependencies`.

### Global CLI

```bash
lattice create
```

### Monorepo Local Script

```bash
pnpm lattice create
```

## `add`

Install components selected by explicit names and/or presets.

Signature:

`lattice add [name...] [--preset <preset...>] [--yes] [--dry-run]`

Behavior:

- Without names/presets, interactive mode prompts for selection.
- With `--yes`, at least one component or preset must be provided.
- Adds required peers and required providers.
- Optional providers are reported in output but not auto-installed.

### Global CLI

```bash
lattice add dialog,toast --preset overlay
lattice add --dry-run --preset form
```

### Monorepo Local Script

```bash
pnpm lattice add dialog,toast --preset overlay
pnpm lattice add --dry-run --preset form
```

## `remove`

Remove installed component packages selected by names/presets.

Signature:

`lattice remove [name...] [--preset <preset...>] [--yes] [--dry-run]`

Behavior:

- Without names/presets, interactive mode prompts from currently installed registry components.
- With `--yes`, names or presets are required.
- Missing requested components are skipped and summarized.

### Global CLI

```bash
lattice remove dialog
lattice remove --preset overlay --dry-run
```

### Monorepo Local Script

```bash
pnpm lattice remove dialog
pnpm lattice remove --preset overlay --dry-run
```

## `upgrade`

Upgrade installed `@lattice-ui/*` packages.

Signature:

`lattice upgrade [name...] [--preset <preset...>] [--yes] [--dry-run]`

Behavior:

- With no selection, interactive mode prompts from installed packages.
- With no selection and `--yes`, all installed `@lattice-ui/*` packages are selected.
- Preserves dependency intent (`dependencies` vs `devDependencies`) during upgrade.
- Missing requested packages are summarized and skipped.

### Global CLI

```bash
lattice upgrade
lattice upgrade --preset form --dry-run
```

### Monorepo Local Script

```bash
pnpm lattice upgrade
pnpm lattice upgrade --preset form --dry-run
```

## `doctor`

Run diagnostics for lockfiles, package-manager alignment, peers, providers, and unknown registry packages.

Signature:

`lattice doctor`

Behavior:

- Warnings: advisory issues (for example optional providers, lockfile drift).
- Errors: required provider violations and other hard-fail conditions.
- Required provider errors throw and terminate command execution.
- Recommendations are output as package-manager-aware commands.

### Global CLI

```bash
lattice doctor
```

### Monorepo Local Script

```bash
pnpm lattice doctor
```

## Common Failure Cases

- `create --yes` without `[project-path]`.
- `create --lint --no-lint` flag conflict.
- `add --yes` or `remove --yes` without explicit selection.
- Unknown component or preset names.

For execution phase output (`Selecting`, `Planning`, `Dry Run`, `Applying`, `Result`, `Next Steps`), see [Execution and Output](/cli/execution-and-output/).
