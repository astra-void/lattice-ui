Source of truth:

- `packages/cli/registry/components.json`
- `packages/cli/registry/presets.json`

## Presets

- `overlay`: `popover`, `tooltip`, `dialog`, `toast`
- `form`: `checkbox`, `radio-group`, `switch`, `text-field`, `textarea`

## Install by Preset

### Global CLI

```bash
lattice add --preset overlay
lattice add --preset form
```

### Monorepo Local Script

```bash
pnpm lattice add --preset overlay
pnpm lattice add --preset form
```

## Install by Explicit Component Names

Supported component names:

- `accordion`, `avatar`, `checkbox`, `combobox`, `core`, `dialog`, `focus`, `layer`, `menu`
- `popover`, `popper`, `progress`, `radio-group`, `scroll-area`, `select`, `slider`, `style`
- `switch`, `system`, `tabs`, `text-field`, `textarea`, `toast`, `toggle-group`, `tooltip`

### Global CLI

```bash
lattice add core,style,system
lattice add select,combobox,menu
```

### Monorepo Local Script

```bash
pnpm lattice add core,style,system
pnpm lattice add select,combobox,menu
```

## Registry Sync Checklist

When registry data changes:

1. Update this page from `components.json` and `presets.json`.
2. Confirm command syntax with `pnpm lattice --help` and examples on [CLI Commands](/cli/commands/).
3. Rebuild docs and validate `/cli/components-and-presets/` renders correctly.
